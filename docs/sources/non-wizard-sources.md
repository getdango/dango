# Non-Wizard Sources: REST API & dlt Native

For APIs and data sources not covered by the built-in wizards, Dango supports two advanced source types.

## REST API

**Best for:** Any HTTP API with JSON responses. No Python required — configure endpoints declaratively.

### Setup

```bash
dango source add
# Select "REST API (Generic)"
# Follow the guided wizard
```

### Configuration

The wizard collects:

| Parameter | Required | Description |
|-----------|----------|-------------|
| `base_url` | Yes | API base URL (e.g., `https://api.example.com`) |
| `endpoints` | Yes | List of endpoint paths and names |
| `auth_type` | No | `bearer`, `api_key`, `basic`, `oauth2_client_credentials`, or `none` |
| `auth_token_env` | No | Environment variable containing the auth token |

### Example: JSONPlaceholder (No Auth)

After running the wizard with `base_url: https://jsonplaceholder.typicode.com` and endpoints `[{"path": "/posts", "name": "posts"}, {"path": "/users", "name": "users"}]`:

```yaml
# sources.yml (generated by wizard)
sources:
  jsonplaceholder:
    source_type: rest_api
    base_url: https://jsonplaceholder.typicode.com
    endpoints:
      - path: /posts
        name: posts
      - path: /users
        name: users
```

### Example: Authenticated API

For an API requiring a Bearer token:

1. Set the token in `.env`:
   ```
   MY_API_TOKEN=your-token-here
   ```

2. Configure with auth:
   ```yaml
   sources:
     my_api:
       source_type: rest_api
       base_url: https://api.example.com/v1
       auth_type: bearer
       auth_token_env: MY_API_TOKEN
       endpoints:
         - path: /customers
           name: customers
         - path: /orders
           name: orders
   ```

### Tips

- The runner transforms wizard endpoint format to dlt's expected format automatically.
- Incremental loading is supported via dlt's pagination and cursor handling.
- For complex pagination or nested resources, consider using dlt Native instead.

## dlt Native Source

**Best for:** Complex integrations requiring custom Python logic — pagination, data transformation, multi-step workflows.

### Setup

```bash
dango source add
# Select "dlt Native Source (Advanced)"
# Choose: File-based, Custom sources dir, or dlt verified source
```

### Three Approaches

#### 1. File-Based Source

Create a Python file with `@dlt.source` and `@dlt.resource` decorators:

```python
# my_source.py
import dlt

@dlt.source
def my_api_source(api_key=dlt.secrets.value):
    @dlt.resource(write_disposition="merge", primary_key="id")
    def items():
        # Your API logic here
        response = requests.get("https://api.example.com/items",
                                headers={"Authorization": f"Bearer {api_key}"})
        yield response.json()

    return items
```

The wizard prompts for the file path, source function name, and registers it in `sources.yml`.

#### 2. Custom Sources Directory

Place source modules in your project's custom sources directory. Dango auto-discovers them.

#### 3. dlt Verified Source

Use any [dlt verified source](https://dlthub.com/docs/dlt-ecosystem/verified-sources) by specifying the module and function name. Install the source's pip dependencies first.

### Tips

- Credentials go in `.dlt/secrets.toml` or `.env` — dlt resolves them automatically.
- Use `write_disposition="merge"` with `primary_key` for incremental/dedup loading.
- Test your source standalone first: `python my_source.py` before registering with Dango.
